Database Models

Major Types of Database Models

  • Relational databases: Organize data into tables with rows and columns (e.g., MySQL, PostgreSQL)
  • NoSQL databases: Store data in formats other than relational tables
    • Document stores (MongoDB, CouchDB)
    • Key-value stores (Redis, DynamoDB)
    • Column-family stores (Cassandra, HBase)
    • Graph databases (Neo4j, ArangoDB)
  • Object-oriented databases: Store data in the form of objects (e.g., ObjectDB)
Key Insight: The choice of database model should align with the specific requirements of your application.

SQL Fundamentals

Basic SQL Query Structure

SELECT column1, column2, ...
FROM table_name
WHERE condition
GROUP BY column_name
HAVING condition
ORDER BY column_name [ASC | DESC]
LIMIT count;

Example: Retrieving Student Information

SELECT s.student_id, s.name, c.course_name, e.grade
FROM students s
JOIN enrollments e ON s.student_id = e.student_id
JOIN courses c ON e.course_id = c.course_id
WHERE c.department = 'Computer Science'
ORDER BY s.name ASC;

Transaction Management

ACID Properties

Atomicity
Consistency
Isolation
Durability
  • Atomicity: Transactions are all-or-nothing operations
  • Consistency: Transactions maintain database integrity
  • Isolation: Concurrent transactions do not interfere with each other
  • Durability: Completed transactions persist even after system failures
